# Glossary of Terms for SOC Analyst Portfolio

This file serves as a quick reference for key terms, acronyms, and concepts encountered throughout this project and the field of cybersecurity. It is designed to aid understanding for both the learner and anyone reviewing this portfolio.

---

## Terms

*   **CLI (Command-Line Interface):** A text-based interface used to interact with computer programs or operating systems by typing commands. (See: Course 4, Module 1)
*   **FHS (Filesystem Hierarchy Standard):** A standard defining the directory structure of Linux operating systems. (See: Course 4, Module 2)
*   **GUI (Graphical User Interface):** A visual interface that uses graphical elements like windows, icons, and menus. (See: Course 4, Module 1)
*   **OS (Operating System):** The core software that manages computer hardware and software resources. (See: Course 4, Module 1)
*   **PAT (Personal Access Token):** A secure alternative to using a password for authenticating to GitHub from the command line.
*   **RegEx (Regular Expressions):** A sequence of characters that defines a search pattern, primarily for use in pattern matching with strings. (See: Course 7, Module 3)
*   **SOC (Security Operations Center):** A centralized unit that deals with security issues on an organizational and technical level.
*   **SQL (Structured Query Language):** A specialized programming language used for managing data held in a relational database management system. (See: Course 4, Module 4)
*   **VM (Virtual Machine):** A virtualized instance of a computer system that runs on top of a hypervisor. (See: Course 4, Module 1)
*   **`venv` (Virtual Environment):** A self-contained directory that holds a specific version of Python and all necessary packages for a project, isolating dependencies.
